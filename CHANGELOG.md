# Хронология изменений

Формат ведения версий: `MAJOR.MINOR.PATCH`.

- `MAJOR` меняется при несовместимых изменениях или крупной перестройке проекта.
- `MINOR` меняется при добавлении новых возможностей.
- `PATCH` меняется при исправлениях, улучшениях интерфейса и небольших правках поведения.

Правило работы: каждое изменение кода фиксируется отдельным коммитом и отправляется в GitHub. Для каждой версии здесь указывается, что добавлено, что изменено/исправлено и зачем это сделано.

## Версия 0.4.3 — 2026-06-26

### Добавлено

- Добавлена карточка пользователя в админ-панели.

- Добавлен список последних заказов пользователя с товаром, количеством, суммой и статусом.
- Добавлена пагинация списка пользователей и кнопочный переход в карточку пользователя.

### Изменено

- Раздел `👥 Пользователи` теперь открывает кнопочный список вместо простого текстового списка.
- Просмотр заказов в админских сервисах приведен к актуальной модели с несколькими выданными кодами.

### Проверка

- `python3 -m compileall app`
- `python3 -m unittest app.tests.test_security_static -v`

## Версия 0.4.2 — 2026-06-26

## Версия 0.4.0 — 2026-06-25

### Добавлено

- Добавлена интеграция платежной системы Platega.
- Добавлено создание внешней ссылки на оплату через Platega из меню покупки.
- Добавлен callback endpoint `/payments/platega/callback` для POST-уведомлений Platega.
- Добавлена проверка callback-заголовков `X-MerchantId` и `X-Secret`.
- Добавлена автоматическая выдача цифровых кодов после статуса Platega `CONFIRMED`.
- Добавлены настройки `PAYMENT_PROVIDER` и `PLATEGA_*` в конфигурацию и `.env.example`.

### Изменено

- Покупка теперь может работать через два провайдера: `telegram` или `platega`.
- При включенной Platega бот запускает HTTP endpoint для приема платежных уведомлений.
- Docker Compose публикует `WEBHOOK_PORT`, чтобы callback можно было подключить через публичный HTTPS-адрес.
- Git-копия синхронизирована с боевой моделью заказов на несколько кодов в одном заказе.

### Исправлено

- Подтянута актуальная модель `Order.issued_items`, чтобы заказы с количеством больше 1 корректно сохраняли и выдавали несколько цифровых товаров.
- Повторные подтверждения платежа не выдают новые коды повторно.

### Проверка

- `python3 -m compileall app`
- `python3 -m unittest app.tests.test_security_static -v`

### Важно для включения

- Platega не включается в боевом `.env` без настоящего API-ключа.
- В кабинете Platega нужно указать публичный HTTPS callback URL вида `https://домен/payments/platega/callback`.

## Версия 0.3.0 — 2026-06-21

### Добавлено

- Добавлена группировка пользовательского списка товаров по подкатегориям.
- Добавлен читаемый экран покупки: товар отображается в тексте с полным названием, ценой и остатком.
- Добавлены короткие кнопки покупки по стране и номиналу, например `🇺🇸 50 USD`.
- Добавлена единая сортировка товаров по номиналу от меньшего к большему.

- Экран покупки больше не использует кнопки вида `1. 100.00 RUB`, чтобы не путать клиента.

- Подсказка в меню покупки изменена на выбор по стране и номиналу.
- Порядок товаров в пользовательском каталоге, покупке и админке приведён к одному правилу.
- Админский список товаров стал компактнее и удобнее для просмотра на мобильном экране.

### Исправлено

- Исправлена нечитаемость длинных кнопок товаров в Telegram.
- Убрана неоднозначность кнопок-стрелок без подписи.
- Устранена путаница между номиналом карты и рублёвой ценой в кнопках покупки.

### Проверка

- `python3 -m compileall app`
- `python3 -m unittest app.tests.test_security_static -v`

## Версия 0.2.0 — 2026-06-18

### Добавлено

- Убрана переменная `TELEGRAM_PROXY_URL` из конфигурации и примера `.env`.
- Бот снова подключается к Telegram API напрямую через стандартную aiogram-сессию.
- Удалена инструкция по proxy из README.
- Добавлена защита от попадания macOS AppleDouble-файлов `._*` и `.DS_Store` в Docker build context.
- Обновлен `DEPLOY_REPORT.md` по фактическому успешному deploy на латвийский VPS.
- Добавлено кнопочное создание и редактирование товаров в админ-панели.
- Добавлены FSM-диалоги для пошагового заполнения товара и кнопки редактирования карточки товара.
- Добавлены audit-safe сервисы для изменения описания и валюты товара.

### Причина

Переезд на новый VPS в Латвии, где ожидается прямой доступ к `https://api.telegram.org` без proxy. Также устранен риск падения Alembic из-за служебных macOS-файлов с нулевыми байтами, зафиксирован результат deploy и улучшена эргономика админки для управления товарами.

## Версия 0.1.1 — 2026-06-17

### Добавлено

- Добавлена переменная `TELEGRAM_PROXY_URL` для исходящих запросов Telegram API через HTTP/HTTPS proxy.
- Бот теперь создает aiogram `AiohttpSession` с proxy из `.env`.

VPS не имеет прямого исходящего доступа к `https://api.telegram.org`, из-за чего polling падал с `TelegramNetworkError: Request timeout error`.
