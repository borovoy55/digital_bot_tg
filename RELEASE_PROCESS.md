# Правила версий и публикации изменений

## Обязательное правило

После каждого изменения кода нужно:

1. Обновить номер версии в файле `VERSION`.
2. Добавить запись на русском языке в `CHANGELOG.md`.
3. Проверить проект доступными быстрыми проверками.
4. Сделать git-коммит с понятным сообщением.
5. Отправить коммит на GitHub в `main`.

## Что писать в `CHANGELOG.md`

Для каждой версии указывать:

- что добавлено;
- что изменено или откорректировано;
- что исправлено;
- какие проверки выполнены.

## Формат версий

Используется формат `MAJOR.MINOR.PATCH`.

- `MAJOR`: крупные несовместимые изменения.
- `MINOR`: новые возможности.
- `PATCH`: исправления, улучшения интерфейса, небольшие корректировки.

## Язык

Хронология изменений, описание версий и пользовательские пояснения ведутся на русском языке.
