# SELF REVIEW

Дата: 2026-06-16

## Итог

Проведена автоматизированная и ручная самопроверка проекта после реализации. Ниже перечислены потенциальные проблемы, найденные во время review, и внесенные исправления.

## Найдено и исправлено

1. Двусмысленная ORM-связь между `orders` и `digital_items`.
   Исправлено: оставлена авторитетная one-to-one связь через `digital_items.order_id`.

2. Статус `error` мог откатываться при исключении “нет кодов после оплаты”.
   Исправлено: ошибка фиксируется в транзакции, исключение поднимается после commit.

3. Проверка charge id могла сравнить `NULL`.
   Исправлено: условия поиска платежа строятся только по непустым charge id.

4. В test Docker image не копировался `requirements-dev.txt`.
   Исправлено: Dockerfile копирует production и dev requirements.

5. Webhook-режим был объявлен, но не запускал сервер.
   Исправлено: добавлен aiohttp webhook runner.

6. В карточке заказа `payment_id` ошибочно отображал ID заказа.
   Исправлено: `payment_id` берется из `order.payments`.

7. Back-кнопка товаров вела к подкатегориям без `category_id`.
   Исправлено: кнопка явно ведет к категориям.

8. Не хватало импорта цифровых товаров из документа.
   Исправлено: `.txt/.csv` файл поддерживается через caption `/admin_upload_items product_id`.

9. Не хватало экспорта цифровых товаров.
   Исправлено: добавлена команда `/admin_export_items`.

10. Не хватало поиска цифровых товаров.
    Исправлено: добавлена команда `/admin_search_items`.

11. Не хватало изменения цифрового товара.
    Исправлено: добавлена команда `/admin_update_item` с проверкой дублей.

12. Не хватало удаления цифрового товара.
    Исправлено: добавлена команда `/admin_delete_item`, статус меняется на `deleted`.

13. Не хватало audit-log для удаления/изменения кодов.
    Исправлено: соответствующие сервисы пишут `audit_logs`.

14. Не хватало изменения цены товара.
    Исправлено: добавлена команда `/admin_update_price` с audit-log.

15. Не хватало изменения статуса заказа.
    Исправлено: добавлена команда `/admin_set_order_status`.

16. Не хватало комментария к заказу.
    Исправлено: добавлена команда `/admin_comment_order`.

17. Не хватало переименования сущностей каталога.
    Исправлено: добавлена команда `/admin_rename`.

18. Не хватало изменения сортировки каталога.
    Исправлено: добавлена команда `/admin_sort`.

19. Не хватало безопасного delete для категорий/подкатегорий/товаров.
    Исправлено: добавлена команда `/admin_delete_entity`, выполняющая soft delete через отключение.

20. Aiogram callback-фильтры были составлены несовместимо с aiogram 3.
    Исправлено: условия объединены через `&`.

21. Ruff auto-fix перевел ORM-аннотации в вид, неудобный для локального Python 3.9.
    Исправлено: ORM-аннотации возвращены к `Optional[...]`/`List[...]`.

22. Ruff auto-fix заменил `timezone.utc` на `datetime.UTC`, недоступный в локальном Python 3.9.
    Исправлено: возвращен `timezone.utc`.

23. `aiohttp` был нужен webhook-режиму, но не был указан в project dependencies.
    Исправлено: добавлен в `requirements.txt` и `pyproject.toml`.

24. Локальный dev-прогон на Python 3.9 требовал pydantic backport.
    Исправлено: `eval-type-backport` добавлен в dev dependencies.

## Проверки

- `python3 -m compileall -q app` — успешно.
- `ruff check .` — успешно.
- `pytest -q` — 8 passed, 4 skipped.
- SQLAlchemy mapper import — успешно.
- SQLAlchemy schema create через SQLite — успешно.

## Почему есть skipped tests

4 PostgreSQL security tests пропущены локально, потому что в этой среде нет Docker и не задан `TEST_DATABASE_URL`. Они готовы к запуску в Docker/profile `test` или на сервере с PostgreSQL.
